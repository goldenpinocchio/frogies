Froggies · 4,032 supply · fc block logic

The pond remembers the block.

Froggies is a Bitcoin-native pond collection built like a poster archive: one hero frog per token, deterministic traits from a 4,032-block fc window, and a visual language that stays lo-fi, editorial, and unmistakably pond-first.

This page is the logic layer for the collection — where the 2 × 2,016 generation split, block-height rarity buckets, and ultra-rare hash rules live before the art is finalized.

Reveal logic

Two generations. 4,032 blocks. No hand-waved rarity.

The new supply works because the fc window naturally splits into 2 generations of 2,016 blocks each. Most visible traits can be assigned from block-height digit buckets; the flashiest ultra-rares stay reserved for Bits / hash so gold and diamond remain truly scarce.

Generation split2,016 / 2,016

The first digit gives a perfect fork: 2xxxxx vs 9xxxxx. The second digit also gives a clean split: x3xxxx vs x6xxxx. Either can anchor the two fc generations.

Sweet spot digitstrait bucketing

The fourth and fifth digits produce the cleanest rarity-sized groups, while the second and third digits support broader common / uncommon splits.

Exact count palettenatural buckets

The window hits useful sizes directly: 1,472, 1,280, 1,216, 1,152, 960, 800, 768, 640, 608, 576, 512, 448, 400, 384, 352, 320, 256, 192, 160, 128, 96, and 64.

Ultra-rare reserveBits / hash

Use block-height digits for body, eyes, mouth, headgear, swamp accessory, and background / aura. Reserve Bits or hash checks for the rarest materials — especially gold and diamond.

Example match · 1,472eyes candidate

pos3 in [0-5] AND pos4 in [0-5] gives an exact 1,472-count bucket.

Example match · 1,280accessory / aura

pos2 in [0-4] AND pos4 in [1-8] gives an exact 1,280-count bucket.

Example match · 64rare floor

pos2 in [8-9] AND pos5 in [4-8] gives a clean 64-count rarity bucket.

Prefix hit · 128special case

The 268 height prefix gives exactly 128, useful for a named swamp detail or limited aura state.
